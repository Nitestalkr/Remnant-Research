# Research Proposals

## Overview

Research proposals are structured hypotheses generated by the GRAO system when gradient analysis indicates a clear direction for investigation. Each proposal represents a concrete research action to take.

## Proposal Format

```json
{
  "proposal_id": "prop_YYYY-MM-DD_NNN",
  "timestamp": "ISO-8601",
  "status": "pending|active|completed|rejected",
  "source_gradients": ["grad_ids"],
  "hypothesis": "clear statement of what to investigate",
  "research_question": "specific question to answer",
  "methodology": "approach to testing",
  "expected_outcome": "what success looks like",
  "confidence": 0.0,
  "priority": "high|medium|low",
  "resources_needed": [],
  "timeline": {
    "start": "ISO-8601",
    "estimated_completion": "ISO-8601"
  },
  "results": {},
  "lessons_learned": []
}
```

## Proposal Generation Criteria

A proposal is generated when:

1. **Gradient threshold** — at least one directional gradient exceeds 0.50 magnitude
2. **Pattern consistency** — pattern observed across ≥3 traces
3. **Novelty check** — not already investigated in active/completed proposals
4. **Resource availability** — can be executed with current capabilities

## Proposal Lifecycle

```
Gradient Signal → Proposal Draft → Confidence Check → Status Decision
     │                                    │
     │                              pending → active (≥0.60)
     │                              pending → rejected (<0.60)
     │
active → completed (results gathered)
active → rejected (pattern decayed)
```
